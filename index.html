AVGVSTO is an advanced encryption suite that binds every encrypted file to a specific USB drive. Without that physical key, decryption is impossible, even with the correct password.

Download GitHub Learn more

Key Features

Hardware binding

The key is derived from your USB's hardware identifier. Even with the password, without that exact drive the files remain unreadable.

AES-256-GCM

Authenticated encryption, the same standard used by banks and governments. Protects both confidentiality and integrity.

Attempt limit

Set a maximum number of decryption attempts. After the limit is reached, the file is locally locked.

Duress password

Two distinct passwords produce two different files (real and decoy). No one can prove which one is real.

Secure delete

The original file is overwritten multiple times before deletion, making recovery impossible on HDDs.

Reset password

If you enable attempt limit, you can set a reset password (stored on the USB) to unlock counters.

How it works

1

Bind your USB

Insert your drive and set it as the authorised hardware key.

2

Encrypt

Choose files or folders, set a password and attempt limit. The original is securely deleted.

3

Decrypt

Plug in the same USB, enter the password. If attempts are exhausted, use the reset password (if configured).

USB device ID

Progressive cooldown

An attacker with the encrypted file but without the USB cannot decrypt. With the USB but without the password neither. If you set an attempt limit and it is reached, the file is locked locally: you can only unlock it with the reset password (stored on the USB).

Frequently Asked Questions

What if I lose my USB drive?

Without the exact USB drive, decryption is impossible. We strongly recommend backing up the encrypted files and storing the USB in a safe place. There is no backdoor.

Can I change the USB binding after encryption?

No, the binding is permanent. You would need to decrypt the file with the original USB and re-encrypt it with the new drive.

What happens when the attempt limit is reached?

The file becomes locked locally. If you configured a reset password (stored on the USB), you can use it to unlock the counter and try again.

Is the duress password really safe?

Yes. The two files (real and decoy) are indistinguishable. There is no way to prove which password unlocks meaningful data.

Can I use any USB drive?

Yes, any USB flash drive or external HDD/SSD should work, as long as it provides a unique hardware identifier. Some very old drives might not work properly.

Does AVGVSTO work on SSD?

Yes, but the secure delete function is less effective on SSDs due to wear leveling. We still overwrite the data, but physical recovery might be possible on some SSDs.

How is the USB identifier retrieved?

AVGVSTO reads the USB's serial number and manufacturer data via the operating system's low-level APIs. This combination is hashed and used as part of the encryption key.

Can I encrypt entire folders?

Absolutely. You can select a folder and AVGVSTO will compress and encrypt its contents into a single .avgvsto file.

Is the software open source?

Yes, AVGVSTO is completely open source under the MIT license. You can inspect, modify, and distribute the code freely.

What is the maximum file size?

There is no hard limit. AVGVSTO uses streaming encryption, so it can handle files up to several terabytes, limited only by available disk space and memory.
