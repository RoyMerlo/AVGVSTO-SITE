AVGVSTO is an advanced encryption suite that binds every encrypted file to a specific USB drive. Without that physical key, decryption is impossible, even with the correct password.

Download GitHub Learn more

Key Features

Hardware binding

The key is derived from your USB's hardware identifier. Even with the password, without that exact drive the files remain unreadable.

AES-256-GCM

Authenticated encryption, the same standard used by banks and governments. Protects both confidentiality and integrity.

Attempt limit

Set a maximum number of decryption attempts. After the limit is reached, the file is locally locked.

Duress password

Two distinct passwords produce two different files (real and decoy). No one can prove which one is real.

Secure delete

The original file is overwritten multiple times before deletion, making recovery impossible on HDDs.

Reset password

If you enable attempt limit, you can set a reset password (stored on the USB) to unlock counters.

How it works

1

Bind your USB

Insert your drive and set it as the authorised hardware key.

2

Encrypt

Choose files or folders, set a password and attempt limit. The original is securely deleted.

3

Decrypt

Plug in the same USB, enter the password. If attempts are exhausted, use the reset password (if configured).

Security model

Algorithm
AES-256-GCM
Key derivation
PBKDF2-SHA256, 1M iterations
Hardware binding
USB device ID
Brute‑force protection
Progressive cooldown

An attacker with the encrypted file but without the USB cannot decrypt. With the USB but without the password neither. If you set an attempt limit and it is reached, the file is locked locally: you can only unlock it with the reset password (stored on the USB).
